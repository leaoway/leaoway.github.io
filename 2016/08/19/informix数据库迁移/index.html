informix数据库迁移

发表于 2016-08-19   |  
  1. 导出数据库 dbexport 库名
  2. 把导出的数据库打成tar包，放到需要搭建新数据库环境的服务器上
  3. 将dbexport.out的第一行{ DATABASE XXXXXXX delimiter | }中的XXXXXXX 改成新建数据库的名称，将XXXXXXXX.exp目录中的XXXXXXXX改为新建数据库的名称，进入XXXXXXXX.exp目录中找到XXXXXXXX.sql，将XXXXXXXX.sql中的XXXXXXXX改为新建数据库名称，并将第一行的 { DATABASE XXXXXXX delimiter | }的XXXXXXXX改为新建数据库的名称。
  4. 进入XXXXXXXX.exp目录，找到XXXXXXXX.sql文件，把文件开头的grant dba to 的用户名，都替换为改为搭建新服务器的登陆用户名，全文件替换，把不存的删掉。
    全替换命令%s/原名/现名/g。
  5. 将无用的unl清空
    搜索unl对应的表在XXXXXXXX.sql中的行数命令 grep -i -n .unl XXXXXXXX.sql
    将    .unl在XXXXXXXX.sql对应的表的row改为0。
  6. 将库导入到需要搭建数据库服务器上
    查看空间onmonitor
    将库导入 dbimport 新建库名 -d 要搭建的空间 -l buffered
